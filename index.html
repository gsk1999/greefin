Batch Scan Model
Tiered (recommended) — uses Haiku first (~$0.003/receipt), escalates to Sonnet only when fields are missing or receipt is complex. ~25× cheaper than Sonnet-only.
Sonnet only — maximum accuracy for every receipt. Best for difficult thermal/faded receipts.
Enhanced Item Lookup
After scanning, uses web search to identify full product names from receipt shortforms (e.g. HNTS PNTBTR 340G → Hunts Peanut Butter 340g). Best for grocery receipts. Adds ~5–10s and uses slightly more API credits.

Minimum Valid Receipt Range
Entries dated outside this range — whether from a scan or typed in manually — will be rejected when you try to save. Catches OCR errors (like reading 2023 instead of 2026) as well as accidentally saving into the wrong month.

Verify Item Names Online
After scanning, searches the web to verify and correct product names — especially useful for thermal receipts with smudged or misread text (e.g. 99 Speedmart, Giant). Runs alongside Enhanced Lookup if both are on. Adds ~5–8s per scan.
🔌 Test Model Availability
Sends a tiny test request to each model you have an API key for, so you can check they're actually responding right now — rather than finding out mid-scan (like when a provider retires a model without warning).

▾ 🏷️ Description: Show "Various" for Paid To
The auto-generated Description follows: Exp Name - Paid To [Paid For], where "Paid To" is normally your Brand field. For categories where the actual vendor changes every time (e.g. Food & Dining — a different restaurant each visit), check the box below to always show "Various" instead, regardless of the specific Brand on that entry.
Entries saved before this feature existed have no Description. This fills it in for any existing entry that's currently blank — entries that already have something in Description (including anything you've manually typed) are left untouched.
Changing a category's "Various" checkbox above only affects future saves and manual edits — it won't retroactively touch entries that already have a Description. Use this if you want every entry regenerated to match your current settings (this WILL overwrite existing Descriptions, including anything you've manually typed).
